# Set 5: Complete Browser Demonstration Guide (Q1 - Q10)

This guide provides the exact info and code needed to demonstrate every practical in Set 5 using only your web browser.

---

## 🔐 Q1: Student Login (JWT + MongoDB)
*   **Concept**: Token-based authentication. The server generates a JWT "passport" after login.
*   **How to Test**:
    1. Open `http://localhost:6001` in your browser.
    2. Open Console (`F12`).
    3. **Register**:
       ```javascript
       fetch('/register', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ username: 'adi', password: '123' }) }).then(r => r.text()).then(console.log);
       ```
    4. **Login**:
       ```javascript
       fetch('/login', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ username: 'adi', password: '123' }) }).then(r => r.json()).then(data => { console.log("Your Token:", data.token); window.jwt = data.token; });
       ```
    5. **Profile**:
       ```javascript
       fetch('/profile', { headers: { 'Authorization': window.jwt } }).then(r => r.text()).then(console.log);
       ```

## 👔 Q3: Employee Portal (RBAC)
*   **Concept**: Role-Based Access Control. Admin sees everyone; User sees only self.
*   **How to Test**:
    1. **Admin**: `http://localhost:6003/employees?role=admin` (Check for a list of 3 employees).
    2. **User**: `http://localhost:6003/employees?role=user&id=102` (Check that only "Employee A" shows).

## 📝 Q4: Notes API (CRUD + Ownership)
*   **Concept**: Data privacy. Notes are linked to a `userId`.
*   **How to Test**:
    1. Open Console on `http://localhost:6004`.
    2. **Create Note**:
       ```javascript
       fetch('/notes', { method: 'POST', headers: { 'Content-Type': 'application/json', 'userid': 'userA' }, body: JSON.stringify({ title: 'A\'s Note', content: 'Secret' }) }).then(r => r.text()).then(console.log);
       ```
    3. **Verify**: Fetch notes with `userid: userA` (it shows) and `userid: userB` (it is empty).

## 🍪 Q6: Session Login (Express-Session)
*   **Concept**: Cookie-based persistence. The browser "remembers" you without a token.
*   **How to Test**:
    1. Visit `http://localhost:6006/login`.
    2. Visit `http://localhost:6006/profile`. It will show your name.
    3. Close the browser tab and reopen it—it will still remember you!
    4. Visit `http://localhost:6006/logout` and try the profile again—it will be gone.
